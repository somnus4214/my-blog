你好，我是 Somnus 👋#

欢迎来到我的小站～
这里是我用来记录学习、折腾、踩坑、思考和成长的地方。

我是一名喜欢技术、喜欢自己动手搭东西的本科生。
平时会接触 人工智能、大模型应用、机器人、计算机视觉、开发工具链 这些方向，也很喜欢研究各种开发环境、部署流程和命令行工具。

对我来说，技术最有意思的地方之一，不只是“结果跑出来了”✨，而是：

  • 为什么它会报错？
  • 为什么这个方案行得通？
  • 为什么换一种方法性能就更好了？
  • 一个项目到底是怎么从 0 到 1 被搭起来的？

我很享受这种不断探索、不断修修补补、最后终于跑通的过程 🚀

我最近在折腾什么 🛠️#

🤖 大模型 / AIGC / Agent#

我对 LLM、RAG、Agent、生成式分析 这些方向很感兴趣，最近也在了解：

  • 大模型应用怎么落地到真实场景
  • Dify 这类工具怎么快速搭建 AI 应用
  • 对话系统、知识库、工作流、智能体这些东西到底是怎么串起来的
  • 微调、数据准备、强化学习这些更底层一点的内容

相比只看概念，我更喜欢一边学一边自己搭点东西。
哪怕只是一个小 demo，只要能跑起来，就很有意思 😎

💻 工程、部署和开发环境#

除了模型本身，我也很喜欢折腾各种工程相关的东西，比如：

  • Ubuntu
  • WSL
  • Netlify
  • 域名解析、博客部署、静态网站托管
  • 各种环境配置、依赖冲突、报错排查

很多人可能觉得配环境很烦，但我其实还挺喜欢这个过程的（笑）
因为每解决掉一个问题，都会有一种“啊，原来这里才是关键”的感觉。

🧠 编程语言与工具#

我平时会接触这些语言和工具：

  • Python
  • C++
  • Rust
  • VSCode
  • Neovim
  • 命令行工具、文本处理工具、效率工具

我很喜欢那些轻量、直接、强大又优雅的工具。
比如会去研究 ripgrep 为什么这么快，也会认真折腾编辑器配置、语言支持、终端体验这些细节。

有时候，工具本身也是一种乐趣 ⚡

🚗 机器人与仿真#

我也接触过 ROS、Gazebo 相关的内容，关注过：

  • 机器人运动仿真
  • TF 广播
  • 编队控制
  • 多机器人系统
  • 机器人程序实现与调试

我很喜欢这种“代码不只是停留在屏幕里，而是对应着一个真实系统逻辑”的感觉。
哪怕是在仿真环境里，也会让人觉得很酷 🤖

👀 计算机视觉 / 三维重建#

除此之外，我对 双目三维重建、配准、非刚性配准 这些内容也有兴趣。
我会关注：

  • 传统视觉方法怎么做
  • 深度学习方法怎么做
  • 两种思路各自适合什么场景
  • 真正写代码的时候流程到底该怎么串起来

这部分内容有时候挺硬核，但也正因为如此，学懂一点点都会很有成就感 📌

我为什么要搭这个博客 ✍️#

这个博客对我来说，不只是一个“个人主页”，更像是一个长期更新的数字笔记本。

我想把这里用来记录：

  • 学习过程中的理解和总结
  • 项目开发里的思路和踩坑
  • 环境配置与部署时遇到的问题
  • 对 AI、工程实践、机器人和视觉方向的一些思考
  • 那些看起来琐碎，但以后一定会感谢自己记下来的东西

比起只展示最后的结果，我更想记录真实的过程。
因为很多时候，真正有价值的，恰恰就是那些“差点没搞定”的瞬间 😂

我的关键词 🏷️#

Python · C++ · Rust · Linux · Ubuntu · WSL · ROS · Gazebo · Neovim · Netlify · LLM · RAG · Agent · Dify · Computer Vision

现在的目标 🎯#

现阶段，我希望自己能继续：

  • 更系统地理解大模型应用开发和工程落地
  • 提升代码能力和项目实现能力
  • 把零散的学习内容慢慢沉淀成自己的知识体系
  • 保持输出，持续记录，不断成长

技术这条路很长，我现在也还在路上。
不过没关系，一边学、一边做、一边记，总会慢慢走远的 🌱

最后 😊#

这里会记录我的学习、思考、尝试，也会保留一些踩坑之后的“惨痛经验”。
